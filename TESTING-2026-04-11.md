# Testing: pi-bg-process-windows

## Build & Install

```bash
cd C:\Users\babys\Documents\code\pi-extensions-dev\pi-bg-process-windows

# Build
bun build ./src/index.ts --outdir ./dist --target node

# Install (copy dist + native binaries)
cp dist/index.js ~/.pi/agent/extensions/pi-bg-process-windows/index.js
cp -r native/ ~/.pi/agent/extensions/pi-bg-process-windows/native/
```

Restart pi or use `/new` to pick up changes.

## Interactive Test Plan

After building and installing, test in a live pi session:

### 1. Quick command (sync: true, default)
```
bash command="echo hello"
```
Expected: Returns "hello" immediately, no backgrounding.

### 2. Auto-background (sync: true, exceeds timeout)
```
bash command="sleep 70 && echo done"
```
Expected: Returns after 60s with PID and log path. `[BG_DONE]` arrives automatically when it finishes.

### 3. PTY mode with peek
```
bash command="sleep 5 && echo 'step 1' && sleep 5 && echo 'step 2'" sync=false
```
Expected: Returns immediately with PID. Use `win_bg_status peek <pid>` to see output as it comes in.

### 4. PTY mode with input
```
bash command="bash -c 'read -p \"Enter name: \" name && echo \"Hello, $name\"'" sync=false
```
Wait for prompt to appear, then:
```
win_bg_status action=input pid=<pid> inputText="TestUser\n"
```
Expected: "Hello, TestUser" appears in output.

### 5. Scroll with peek offset
```
bash command="for i in $(seq 1 100); do echo \"line $i\"; sleep 0.1; done" sync=false
```
Then:
```
win_bg_status action=peek pid=<pid> lines=10 offset=20
```
Expected: Shows 10 lines starting 20 lines before the end of the buffer.

### 6. Process list and stop
```
win_bg_status action=list
win_bg_status action=stop pid=<pid>
```

### 7. [BG_SILENT] notification
Start a command that produces no output for 30s+:
```
bash command="sleep 35 && echo finally" sync=false
```
Expected: After 30s, `[BG_SILENT]` notification arrives with recent output.

### 8. /win_tasks command
```
/win_tasks
```
Expected: Interactive TUI showing running processes with scroll and kill capability.

## Key Findings from Testing

### taskkill hangs from Node/Bun
`taskkill /T /F /PID` hangs when called from Node/Bun targeting child processes. Use `process.kill(pid, "SIGKILL")` + PowerShell orphan cleanup instead.

### deliverAs: "nextTurn" doesn't auto-notify
Using `deliverAs: "nextTurn"` in `sendMessage` causes notifications to wait for the next user prompt. Must use `deliverAs: "followUp"` for automatic delivery.

### PTY paths work correctly
Both `C:\Program Files\Git\bin\bash.exe` (backslashes) and `C:/Program Files/Git/bin/bash.exe` (forward slashes) work correctly with node-pty on Windows.

### pi -p may appear to hang
Running `pi -p "echo hello"` can be slow due to extension loading overhead. This is a startup delay, not a hang.
